    Change or Localize Text in WHMCS

    task
    You can change the English text customers see in the WHMCS plug-in, and you can also localize this text into other languages.

    About Changing or Localizing Text

    Default text for the Weebly WHMCS plug-in is stored in the ./lang/weebly_lang_english.php file. You can’t directly change values in this file (for example, if you want to change the English value) as your changes would be overwritten with the next upgrade of WHMCS. Instead, for both changing the English, or for creating a localized version in another language, you download a copy of this file (below) and store the copy in an override directory, providing the overritten English or the translated values. Overritten English values will be displayed the next time the customer views the page containing the variable. For translations, when the customer chooses a language, WHMCS uses the variable in that language’s override file instead of the default English text.

    WHMCS supports the following translations:

    • Arabic
    • Azerbaijani
    • Catalan
    • Croatian
    • Czech
    • Danish
    • Dutch
    • English
    • Estonian
    • Farsi
    • French
    • German
    • Hebrew
    • Hungarian
    • Italian
    • Norwegian
    • Portuguese-br
    • Portuguese-pt
    • Romanian
    • Russian
    • Spanish
    • Swedish
    • Turkish
    • Ukranian

    For example, if you were creating a Spanish translation, you might change: $_LANG['WEEBLYCLOUD_PANEL_GETSTARTED'] = "Get Started";

    to:

    $_LANG['WEEBLYCLOUD_PANEL_GETSTARTED'] = "Empezar";

    Be sure to thoroughly test your translations to make sure the characters display properly and the text boxes are sized properly.

    Download This File to Begin

    Download the weebly_lang_english.php file and save it to a local directory. You will need to copy it and save the variations to your WHMCS overrides directory. Follow the instructions below, based on whether or not you have existing WHMCS translations.

    If You Do Not Have Existing WHMCS Language Override Files

    If you have never created localized versions of files for WHMCS, then you’ll need to create the expected directory and override files:

    1. Create a directory named overrides as a child to the <WHMCS_Install_Directory>/lang/ directory.

    2. Copy the weebly_lang_english.php file and paste it to the <WHMCS_Install_Directory>/lang/overrides/ directory, renaming the file to reflect the language, from the list above.

      For example, if you were going to create a Spanish version, you would rename the file to spanish.php. If you were overriding the default English values, then keep the same name, just make sure it’s stored in the overrides directory and that you’re not replacing the original.

    3. Replace the strings for each variable with strings in your desired language. Only change the strings inside the quotes (‘…’). Do not change the variable names.
    Note: Do not change any of the code above /* WeeblyCloud language variables */. Changing that code will result in server errors.

    If You Already Have Language Override Files

    If you already have localized files in the overrides directory, then you need to append the Weebly variables to those and provide translations.

    1. Open the weebly_lang_english.php file and copy everything from /* WeeblyCloud language variables */ to the end of the file. These are all the Weebly variables.
    2. Open your existing language file(s) in the overrides directory and and paste in the Weebly variables to the end of your file, being careful to not overwrite any existing variables.
    3. Replace the strings for each variable with strings in your desired language. Only change the strings inside the quotes (‘…’). Do not change the variable names.
