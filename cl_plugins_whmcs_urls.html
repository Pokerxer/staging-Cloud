    Set up Site Provision Defaults and URLs in WHMCS

    task
    The final step in WHMCS configuration is set provisioning defaults, a billing url (if using cPanel) and an upgrade URL.

    To finish configuration, click the Account Details link in the left hand side.

    ​ The first two options allow you determine the default behavior when a user provisions a new Weebly site. Select Show all Weebly plans to select if you want your customer to be able to choose a Weebly plan as their site is provisioned. Select Automatically assign this plan if you always want a specific plan assigned by default.

    The final setup step is assigning the billing url and upgrade url. The billing url is only relevant when using our cPanel plugin. If you don’t use the cPanel plugin provided by Weebly, then you can skip to the upgrade url.

    Billing Url

    The WHMCS module provides a billing url for you to use in conjunction with our cPanel plugin. This should be set to something like https://Domain_Url/whmcs/weebly_billing.php?type=billing. Domain_Url is the domain of your WHMCS installation. You can learn more about our cPanel billing integration here, but all you really need to do is set the url here.

    Upgrade Url

    The WHMCS module provides an upgrade url that is presented to your users when an upgrade is triggered in the Weebly Editor. This url will send the user to WHMCS and direct them through an upgrade. You can learn more about the Weebly Cloud upgrade process here.

    Once you have set everything, click Save Account Details. From this page you can also update both the billing and upgrade urls stored in Weebly Cloud. When there is a url mismatch red text will appear along with a sync button. This button will update your Weebly Cloud account. This avoids you needing to set these values in Cloud Admin.
