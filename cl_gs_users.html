    Create and Manage Users

    task
    Before you create a site, you must create a user that will be the owner of that site. There are three ways to create users - using Cloud Admin, using the Cloud API, or using one of the plug-ins.
    Note: All sites associated with a user have access to each site’s uploaded files (such as images, pdfs or other documents), and themes. If you need to sites to be completely separate, then you need to create them under separate users.

    Use the Cloud Admin to Create and Manage Users

    Note: Using Cloud Admin to work with users and sites is not available to all Weebly installations. If you do not have this functionality and want more info, contact your Weebly business development sales rep.

    You can create users by clicking the Add User button on the Users tab of Cloud Admin.

    ​ You must provide a name and an email address, and that email must be unique to Weebly. Upon creation, a user ID is associated with the new user.

    Each user has a Manage page that displays user information, such as their location, operating system, sign up date, last log-in, as well as a list of all sites they own and all sites they contribute to. You can do the following from this page:

    • Edit their personal info
    • Reset their password
    • Send an invite email
    • Turn test mode on/off
    • Disable the user
    • Add a site
    • View activity logs
    • Create a support request
    • Add notes
    Tip: If you want to restrict access for a user, then create them as a contributor to a site. Contributors can be configured so that they can’t do things like publish a site or change a theme.

    Use the API to Create and Manage Users

    You can create users programmatically using the User API. You need to include a unique (to Weebly) email address in your POST request. An error will be thrown if an email address is provided that is already in use. A user id (ex. 32134607) is returned upon successful creation of a user. This user id should be saved to your system of record as you’ll want to use it to maintain the user’s account (not necessary if you use the WHMCS or APS2 plug-ins)​.

    Once created, you can use the API to modify the email, language, and test mode. You can also enable/disable the user from being able to use the Weebly editor, and generate a one-time link that directs the user to the last site that was modified in the account. ​

    Use Plug-ins to Create Users

    When using plug-ins, users are created automatically when sites are created. For example, when a user visits the cPanel and clicks Weebly and creates a site, the plug-in automatically creates a user that’s associated with that site. For more information, see the Customer Guide for each individual plug-in.
