    WHMCS Customer Guide

    concepts
    Learn how your customers with work with WHMCS

    WHMCS Standalone

    In a standalone integration everything takes place in WHMCS. The user logs in to WHMCS and under Order New Service they select the Weebly Plans option. They will see all Weebly plans configured in your WHMCS install. Shown below is a default installation which offers all 4 Weebly plans (Free, Starter, Pro and Business).

    The user selects which WHMCS Weebly plan they’d like to create. If you offer both monthly and annual pricing, then this will display in the dropdown. The user must provide a domain name for their Weebly website. They enter this in the Additional Required Information section.

    ​ When the user clicks Add to Cart they are prompted to enter their payment information (or use an existing payment method). At this point the order is created in WHMCS. Once the payment is confirmed the Weebly website is created. Payment confirmation is automatic by default and doesn’t require any manual intervention by the admin.

    Once the order has gone through, the user sees an Active status under My Services. Clicking the View Details button takes the user into an order details page. An Edit Site button takes the user into the Weebly editor where they can start working on their website.

    From here, the user can build the website.

    WHMCS with cPanel

    The second type of integration is combining our WHMCS module with our cPanel plugin. As many service providers use both WHMCS and cPanel we’ve tried to make it as seamless as possible to use the products together. Everything stated in the WHMCS standalone guide works in an integration with both WHMCS and cPanel.

    Pictured below is what a user sees when they first click the Weebly icon in cPanel. All the available domains are listed in the dropdown. If a domain is not listed, it is already in use on the Weebly platform or contains the word “Weebly.”

    When the user clicks the Add button they are directed to the billing url, as set in your Cloud Admin account or directly in WHMCS.

    The user can select any Weebly plan you have made available in WHMCS. Once the payment is confirmed in WHMCS the user has access to their newly created Weebly site. Access is available in either WHMCS or cPanel.

    When the user returns to cPanel they are taken to the User Portal with a new Edit Site button next to the site they just created. This button takes them directly into the Weebly editor where they can update their site.

    In WHMCS the Edit Site button is found under My Services. Clicking on the View Details button takes the user to an order details page. This page provides the Edit Site button.

    WHMCS Upgrades

    One of the big advantages to using our WHMCS module is the tight integration in the event a user upgrades while using the Weebly editor. The editor determines if a user has access to a given feature by the Weebly plan assigned to the site. If they try to use a feature not offered by their current plan they are prompted to upgrade. The screenshot below shows a user trying to access the HD Video element when they don’t have access to it.

    The modal window below is what is displayed to the user when an upgrade is required to use a certain feature. They can opt to Try it out without upgrading; however, it will not display on the published site.

    When they click Upgrade Now an upgrade window is opened. This is known as the upgrade_url. You can learn more about upgrades here. The good news you shouldn’t have to do much because upgrades are taken care of by the WHMCS addon. Shown below is the WHMCS upgrade window sitting on top of the Weebly editor. The user is prompted for all potential upgrades that will satisfy the specific upgrade.

    After the user chooses which product they’d like to upgrade to they are sent to a summary screen (shown below). After this screen they are prompted for payment method. Once the payment goes through they upgrade window will close and the Weebly editor will refresh automatically. At this point the user will have access to the feature since they have upgrade their site.

    WHMCS Cancellations

    In the event a user (or WHMCS admin) wants to cancel a subscription, the Weebly website will be deleted. This means the user will no longer have access to the site in the Weebly editor. A user can initiate a Request Cancellation within WHMCS under the Management Actions tab of their service details page.

    A WHMCS administrator can also trigger a termination by clicking the Terminate button within WHMCS. If you’re using cPanel with WHMCS, terminate will remove the associated FTP account/credentials, and delete the Weebly site. If you’re using WHMCS standalone, terminate will delete the Weebly site.
